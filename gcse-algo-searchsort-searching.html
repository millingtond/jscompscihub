Introduction to Searching Algorithms

Searching is a fundamental operation in computer science. We often need to find a specific item within a collection of data (like a list or array). An algorithmA finite sequence of well-defined, computer-implementable instructions, typically to solve a class of specific problems or to perform a computation. is a set of step-by-step instructions to solve a problem, and searching algorithms provide these instructions for finding data.

The efficiency of a search can dramatically impact performance, especially with large datasets. Today, we'll explore two common searching algorithms: Linear Search and Binary Search. We'll look at how they work, when to use them, and their respective pros and cons.

Understanding these algorithms helps us appreciate how computers efficiently manage and retrieve information. Hover over keywordsThis is an example tooltip! for definitions.

Linear Search

The One-by-One Approach

Linear SearchA searching algorithm which looks at every element in turn until it finds the target, it is slow but works even on unsorted data. (also known as a sequential search) is the simplest search algorithm. It checks each element in a list, one by one, in sequence, until the desired item is found or the end of the list is reached.

How it Works (Operation):

  1. Access the first value/element in the list.
  2. Compare this value to the search value (the target item).
  3. If it does not equal the search value:
    • Move to the next element in the list (increment a pointerA variable or index that indicates a specific position or element within a data structure (like a list or array) during a search. or index).
    • Repeat step 2.
  4. If it does equal the search value:
    • The item is found! Return true, print "Item found", or return the item's position.
  5. If the end of the list is reached and the item has not been found, then the item is not in the list.

Advantages of Linear Search:

  • Simple to understand and implement.
  • Works on unsorted lists (no pre-processing needed).
  • Efficient for very small lists.
  • Can be useful if the item is likely to be near the start of the list.

Disadvantages of Linear Search:

  • Very inefficient for large lists (slow).
  • On average, it checks half the list. In the worst case, it checks every item.

Binary Search

The Divide and Conquer Approach

Binary SearchA searching algorithm that divides the search space in half each time until it finds the target. Requires the array to be sorted. is a much more efficientA measure of how well an algorithm performs in terms of time (speed) and space (memory usage). Binary search is generally more efficient than linear search for large, sorted lists. searching algorithm, but it has one crucial requirement: the list must be sortedA list or array where elements are arranged in a specific order (e.g., ascending or descending). Essential for binary search. (e.g., alphabetically or numerically).

It works by repeatedly dividing the search spaceThe set of data or range of values within which a search algorithm looks for a target item. in half. If the value of the search key is less than the item in the middle of the interval, narrow the interval to the lower half. Otherwise, narrow it to the upper half. Repeatedly check until the value is found or the interval is empty.

How it Works (Operation):

  1. Start with `low` pointer at the beginning of the list and `high` pointer at the end.
  2. While `low` is less than or equal to `high`:
    • Calculate the `mid` point: `mid = floor((low + high) / 2)`.
    • Compare the item at `mid` with the search value (target).
    • If middle value equals search value: Item found! Return true/position.
    • If middle value is smaller than search value: The target must be in the upper half. Discard the lower half by setting `low = mid + 1`.
    • If middle value is larger than search value: The target must be in the lower half. Discard the upper half by setting `high = mid - 1`.
  3. If `low` becomes greater than `high`, the search space is empty, and the item is not in the list.

Note: If the list size becomes 1 and the search value does not match that item, the value is not in the list.

Advantages of Binary Search:

  • Much faster/more efficient than linear search for large, sorted lists.
  • Reduces the search space by half with each comparison.
  • Fewer comparisons needed on average and in the worst case for large lists.

Disadvantages of Binary Search:

  • Requires the list to be sorted first. Sorting itself takes time.
  • More complex to implement than linear search.
  • Not efficient for small lists (linear search might be faster due to overhead).
  • Direct access to the middle element is needed (works well with arrays, less so with linked lists).

Real-World Context

Searching algorithms are used constantly in the digital world:

  • Linear Search:
    • Finding an item in a small, unsorted shopping list.
    • Checking if a specific username is already taken during signup (if the list of usernames isn't massive or is checked infrequently).
    • Simple "find" features in text editors for small documents.
  • Binary Search:
    • Looking up a word in a physical dictionary (you open to the middle).
    • Finding a contact in your phone's contact list (which is usually sorted).
    • Database lookups on indexed (sorted) fields.
    • Guessing games like "I'm thinking of a number between 1 and 100."
    • Version control systems (like Git) use variations to efficiently find changes between versions.

Search Algorithm Myth Busters!

Myth 1: "Binary search is ALWAYS faster than linear search."

Also, linear search doesn't require the list to be sorted, and sorting takes time.

Myth 2: "You can use binary search on any list of numbers."

Reality: This is a critical misunderstanding! Binary search absolutely REQUIRES the list to be sortedA list or array where elements are arranged in a specific order (e.g., ascending or descending). If you try to use binary search on an unsorted list, it will likely give incorrect results or fail to find items that are present because its logic relies on the ordered nature of the data to discard halves of the search spaceThe set of data or range of values within which a search algorithm looks for a target item..

1. Describe the steps involved in a linear search algorithm to find an item in an array. [3 marks]

Mark Scheme:

  • Start at the first element of the array / Access the first value. (1 mark)
  • Compare the current element with the target item. (1 mark)
  • If they match, the item is found (and stop/return). If not, move to the next element and repeat the comparison. (1 mark)
  • (Allow: If the end of the array is reached and no match, the item is not present.)

2. Explain why a list must be sorted before a binary search can be performed successfully. [2 marks]

Mark Scheme:

  • Binary search works by comparing the target with the middle element of the current search space. (1 mark)
  • Based on this comparison, it discards half of the search space (either the lower or upper half) because it knows the target cannot be in that half if the list is sorted. If unsorted, this assumption is invalid. (1 mark)

3. Give one advantage of using a binary search over a linear search, and one disadvantage. [2 marks]

Mark Scheme:

  • Advantage: Binary search is much faster/more efficient for large lists. (1 mark)
  • Disadvantage: Binary search requires the list to be sorted first (which takes time). (1 mark)
  • (Allow other valid points, e.g., Adv: Fewer comparisons. Disadv: More complex to implement.)

Extension Activities

1. Time Complexity (Big O Notation - Conceptual)

Computer scientists use "Big O notation" to describe the efficiencyA measure of how well an algorithm performs in terms of time (speed) and space (memory usage). Binary search is generally more efficient than linear search for large, sorted lists. of algorithms as the input size grows.

Research Task: Conceptually, what does O(n) (Linear Search worst case) and O(log n) (Binary Search worst case) mean in terms of how many operations are needed as the list size (n) increases? Which one grows much slower for large 'n'?

2. Interpolation Search

For uniformly distributed sorted data, an interpolation search can be even faster than binary search. It tries to guess where an item might be based on its value relative to the range of values in the list (like how you'd guess a page number in a dictionary for a word starting with 'M').

Research Task: Briefly explain the basic idea behind interpolation search. What are its advantages and when might it outperform binary search?

3. Hashing and Hash Tables

For very fast lookups (often close to O(1) - constant time on average), hash tables are used. They use a "hash function" to compute an index directly from the data item itself.

Research Task: What is a hash function? How do hash tables allow for potentially very fast searching? What is a "hash collision" and how might it be handled?
