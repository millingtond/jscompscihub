Introduction to Network Performance

Network performance isn't just about how "fast" your internet feels; it's a combination of several factors that determine how quickly and reliably data can be sent and received. Whether you're gaming online, streaming a movie, or just browsing the web, these factors are constantly at play.

In this lesson, we'll break down the key elements that influence how well a network performs, from the fundamental capacity of the connection (bandwidthThe maximum amount of data that can be transmitted over a network connection in a given amount of time, usually measured in bits per second (bps).) to the delays involved in data travel (latencyThe delay before a transfer of data begins following an instruction for its transfer. Often referred to as 'ping'.), and the impact of other users and the physical environment.

Real-World Context

  • Online Gaming: Low latencyThe delay before a transfer of data begins following an instruction for its transfer. Often referred to as 'ping'. is critical for responsive gameplay. High bandwidth helps download game updates quickly.
  • Video Streaming (Netflix, YouTube): Requires sufficient, consistent bandwidthThe maximum amount of data that can be transmitted over a network connection in a given amount of time, usually measured in bits per second (bps). to avoid buffering. Latency is less critical than for gaming but still matters for starting streams.
  • Office Networks: Performance is affected by the number of users, the type of tasks (e.g., large file transfers vs. email), and the quality of network hardware (switches, routers, cabling).
  • Choosing an ISP: Advertised speeds are usually maximum bandwidthThe maximum amount of data that can be transmitted over a network connection in a given amount of time, usually measured in bits per second (bps).. Real-world performance also depends on factors like network congestion in your area and the ISP's overall network capacity.

Network Performance Myth Busters!

Myth: "More bandwidth always means faster internet for everything I do."

Reality: While high bandwidthThe maximum amount of data that can be transmitted over a network connection in a given amount of time, usually measured in bits per second (bps). is great for downloading large files or streaming high-quality video, it doesn't always translate to a "snappier" experience for tasks sensitive to latencyThe delay before a transfer of data begins following an instruction for its transfer. Often referred to as 'ping'., like web browsing (initial page load) or online gaming. If your latency is high, even with massive bandwidth, these activities can still feel slow.

Myth: "Wi-Fi is always much slower than a wired Ethernet connection."

Reality: Modern Wi-Fi standards (like Wi-Fi 6/6E) can offer very high speeds, sometimes comparable to or even exceeding older Ethernet standards for typical home use. However, Wi-Fi is more susceptible to interferenceDisturbances that can corrupt or disrupt data signals during transmission, e.g., from other electronic devices or physical obstructions. and signal degradation due to distance and obstacles, which can reduce its effective performance. Wired connections are generally more stable and less prone to these issues.

Myth: "A few lost packets don't really matter for network speed."

Reality: When data packets are lost or corrupted (high error rateThe frequency at which errors occur in data transmission over a communication channel.), protocols like TCP require them to be retransmitted. These retransmissions take extra time and consume additional bandwidth, effectively slowing down the overall data transfer and increasing perceived latency.

Mark Scheme:

  • Increased number of users means the available bandwidth is shared among more devices. (1 mark)
  • This can lead to slower speeds for each user / increased latency / network congestion / devices having to wait longer to transmit. (1 mark)
  • (Allow: More users can lead to more data collisions (especially on older hub-based or half-duplex wireless networks), requiring retransmissions.)

2. A company is experiencing slow file transfers. Identify two different factors that could be causing this poor network performance and suggest a possible solution for each. [4 marks]

Mark Scheme (1 mark for factor, 1 for linked solution, for two factors):

  • Factor: Insufficient bandwidth. (1 mark)
    Solution: Upgrade to a higher bandwidth connection / ISP plan / install better quality network cabling (e.g. fibre optic). (1 mark)
  • Factor: Too many users / high network traffic. (1 mark)
    Solution: Implement Quality of Service (QoS) to prioritise traffic / segment the network / upgrade network hardware (e.g. faster switches/routers) / schedule large transfers for off-peak hours. (1 mark)
  • Factor: Poor transmission media / high interference. (1 mark)
    Solution: Switch from Wi-Fi to wired Ethernet / upgrade to better shielded cables / move Wi-Fi router away from sources of interference / use a different Wi-Fi channel. (1 mark)
  • Factor: High latency. (1 mark)
    Solution: Choose servers closer geographically (if applicable) / optimise network routing / reduce number of hops. (1 mark)
  • Factor: High error rate. (1 mark)
    Solution: Replace faulty cables or network hardware / reduce interference. (1 mark)

Key Takeaways

  • Network performance is influenced by multiple interconnected factors.
  • BandwidthThe maximum amount of data that can be transmitted over a network connection in a given amount of time, usually measured in bits per second (bps).: Higher bandwidth allows more data to be transferred simultaneously, generally leading to faster downloads/uploads.
  • Number of Users/Devices: More active users sharing the same bandwidth can lead to slower performance for everyone due to congestionA state where a network is carrying so much data that its quality of service deteriorates, leading to slowdowns and delays..
  • Transmission Media: Wired connections (Ethernet, Fibre Optic) are often faster and more reliable than wireless (Wi-Fi), which is prone to interferenceDisturbances that can corrupt or disrupt data signals during transmission, e.g., from other electronic devices or physical obstructions..
  • Error Rate & Interference: Physical obstructions, other electronic devices, or faulty hardware can cause data corruption, leading to retransmissions and slower effective speeds.
  • LatencyThe delay before a transfer of data begins following an instruction for its transfer. Often referred to as 'ping'.: The delay in data travel time, critical for real-time applications like gaming and video calls. Distance and network hops contribute to latency.
