Understanding Software Permissions: Licences

OCR GCSE Computer Science (J277) - Focusing on Open Source and Proprietary Software

Starter Activity (5 mins)

Permissions & Software...

Lesson Outcomes

By the end of this lesson, you should be able to:

  • Define what a software licence is and its purpose.
  • Distinguish between proprietary software and open source software.
  • Describe characteristics of proprietary licences.
  • Describe characteristics of open source licences.
  • Explain what source code is in this context.
  • Define bespoke software.
  • Understand key legal issues when choosing software licences.

Introduction: What are Licences?

A software licence is a legal document that governs the use and/or redistribution of software. It grants the licensee (the user) permission to use the software under certain conditions set by the licensor (the copyright holder, usually the developer or publisher). A key purpose is to make sure the creators get paid or acknowledged for the work they did.

Without a licence, using software would generally be a breach of copyright law, as software is a creative work. Licences can range from very restrictive to very permissive.

We'll explore different types of software licences, focusing on proprietary and open source, and discuss related concepts and legal considerations. Key terms are in bold.

With proprietary software, the licence restricts the copying, modifying, and distribution of the software without the explicit permission of the copyright owner. The source code is typically not made available to users; only the compiled (machine-readable) code is published.

Benefits to Manufacturer: Helps protect their intellectual property, allows them to control development, and often enables them to charge for the software.

Quick Check: True or False? "Users of proprietary software usually have access to its source code."

Open source software is released under a licence that allows users more freedom. Key characteristics include:

  • The source code is distributed with the software (or made easily available).
  • The customer can modify the source code.
  • The customer can redistribute the source code and modified versions, often under conditions (e.g., same licence).
  • Can be free of charge or paid; the "open" refers to freedom and access.

Quick Check: Can open source software be sold for a price?

Freeware: Software available at no cost, but usually proprietary (source code not available, cannot modify/redistribute). Restrictions like "personal use only" may apply.

Shareware: Distributed free on a trial basis. Payment may be needed later for full features or continued use. May expire or have limited functionality until purchased.

Quick Check: Is Shareware always completely free to use indefinitely?

The source code is the human-readable set of instructions (e.g., written in Python, Java, C++) that define how a software application functions. It's made public for open source software but remains a secret in proprietary software, where users usually only get the compiled, machine-executable version.

Quick Check: Proprietary software licences typically grant access to the software's source code. True or False?

Bespoke software (also known as custom software) is specially designed and written for a single organisation or group of users to meet their specific requirements. It is tailored to their unique workflows and needs, unlike off-the-shelf software designed for a mass market.

Quick Check: Is Microsoft Word an example of bespoke software for most users?

Choosing and using software involves several legal considerations:

  • You must abide by the software licence terms.
  • For open source software, modifications may need to be shared back, and contributors credited.
  • Off-the-shelf (proprietary) software must be legally purchased.
  • All software used must enable compliance with laws like the Data Protection Act.

Quick Check: If you modify open source software, you never have to share your changes. True or False?

Task 4: Exam Practice Questions

Apply your knowledge to these exam-style questions.

1. Explain two key characteristics that distinguish open source software from proprietary software. [2 marks]

Mark Scheme (Any two from, 1 mark each):

  • Open source software's source code is publicly available/distributed, whereas proprietary software's source code is usually kept secret.
  • Open source software licences grant users the freedom to modify and redistribute the software, whereas proprietary software licences typically restrict this.
  • Open source software often emphasizes community collaboration, while proprietary software is controlled by a single entity.
  • Open source users may have an obligation to share modifications (copyleft), which is not a feature of proprietary software.

2. A small bakery commissions a software developer to create a unique online ordering system just for their business. What type of software is this an example of? [1 mark]

Key Takeaways

  • A software licence dictates terms of use for software, protecting creators and defining user permissions.
  • Proprietary software typically has restricted use, modification, and distribution, with secret source code.
  • Open source software provides source code access and freedoms to use, study, modify, and distribute, often with conditions.
  • Freeware is free to use but often proprietary; Shareware is try-before-you-buy.
  • Bespoke software is custom-made for specific users.
  • Choosing a licence involves legal issues such as abiding by terms, costs, modification rights, and data protection compliance.

Extension Activities

1. Investigate a Specific EULA

Find the End User Licence Agreement (EULA) for a piece of proprietary software you use (e.g., a game, OS, app).

Research Task: Summarize 2-3 key restrictions stated in its EULA. What are you NOT allowed to do?

Key areas to look for in a EULA:

  • Restrictions on copying or number of installations.
  • Prohibitions on reverse engineering, decompiling, or modifying the software.
  • Limitations on transferring the licence to someone else.
  • Clauses on data collection and privacy by the software vendor.
  • Disclaimers of warranty and limitations of liability.

2. The GPL and "Copyleft"

The GNU General Public License (GPL) is a well-known open source licence with a "copyleft" provision.

Research Task: What does "copyleft" mean? How does the GPL ensure that software (and its modifications) remains open source?

Key Considerations for GPL/Copyleft Research:

  • Copyleft: If you distribute modified versions of GPL software, those versions must also be licensed under the GPL (or compatible terms).
  • This ensures that the software remains free and open, preventing it from becoming proprietary.
  • The "four essential freedoms" of free software.
  • Contrast with permissive licences (like MIT) which don't have this strong reciprocal sharing requirement for modified versions.

3. Business Use of Open Source

Many large companies (e.g., Google, Microsoft, IBM) use and contribute to open source software.

Research Task: Why would a commercial company choose to use or even develop open source software? What are the benefits for them?

Key Considerations for Business Use of OSS Research:

  • Cost savings (often no direct licence fees).
  • Access to a large community for support and development.
  • Ability to customize and adapt the software to their needs.
  • Transparency and security (source code can be audited).
  • Attracting talent (developers often like working with OSS).
  • Building industry standards or ecosystems around their technology.
  • Offering commercial services or products around an open source core.
