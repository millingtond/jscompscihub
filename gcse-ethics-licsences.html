Introduction: What is a Software License?

When you use a piece of software, you don't usually own the software itself. Instead, you are granted permission to use it under certain terms and conditions. This permission is given through a software licenseA legal instrument governing the use or redistribution of software. It grants the user permission to use the software under specified conditions..

A software license is a legal agreement between the software creator (or copyright holder) and the user. It defines what the user can and cannot do with the software, such as how many computers it can be installed on, whether it can be modified, and if it can be shared with others.

Licenses are crucial because software is a form of intellectual property (IP)Creations of the mind, such as inventions; literary and artistic works; designs; and symbols, names and images used in commerce. Protected by patents, copyright and trademarks., protected by laws like the Copyright, Designs and Patents Act (CDPA)UK law protecting intellectual property rights for creators of original literary, artistic, musical, dramatic works, software, films, and sound recordings.. The license ensures that creators get paid or acknowledged for their work and helps prevent unauthorized use or distribution.

There are two main categories of software licenses: Open SourceSoftware with source code that anyone can inspect, modify, and enhance. It's often distributed with a license that grants users the rights to use, study, change, and distribute the software and its source code to anyone and for any purpose. and ProprietarySoftware that is owned by an individual or a company (usually the one that developed it). There are almost always major restrictions on its use, and its source code is almost always kept secret. Also known as closed-source software.. We'll explore these in detail. Open Source Software

Characteristics

Open Source softwareSoftware with source code that anyone can inspect, modify, and enhance. It's often distributed with a license that grants users the rights to use, study, change, and distribute the software and its source code to anyone and for any purpose. is software for which the original source codeThe human-readable instructions written by programmers in a high-level programming language. This is what is modified in open source software. (the human-readable instructions written by programmers) is made freely available and may be redistributed and modified.

Key Characteristics:

  • Source code is distributed with the software: Users can view and study how the software works.
  • Users can modify the source code: This allows for customization, bug fixing, and adding new features.
  • Users can redistribute the source code (and modifications): Often, this must be done under the same or a compatible open source license.
  • Can be free of charge or paid: "Open source" doesn't necessarily mean "free of cost," though much of it is.
  • Relies on community collaboration for development and support.
  • Examples: Linux Operating System, Apache Web Server, Mozilla Firefox browser, GIMP image editor, LibreOffice suite.

Proprietary Software (Closed Source)

Characteristics & Examples

Proprietary softwareSoftware that is owned by an individual or a company (usually the one that developed it). There are almost always major restrictions on its use, and its source code is almost always kept secret. Also known as closed-source software., also known as closed-source software, is software that is owned by an individual or company (usually the developer). Its use is governed by a license that typically restricts copying, modification, and distribution.

Key Characteristics:

  • Source code is NOT made available: Users receive only the compiled, executable code.
  • License restricts use: Users typically cannot legally copy, modify, or redistribute the software without explicit permission.
  • Often requires payment for a license (though some proprietary software can be "freeware").
  • Development and support are usually handled by the owning company.
  • Examples: Microsoft Windows, Microsoft Office, Adobe Photoshop, most commercial video games.

Benefits to the Manufacturer/Developer:

  • Protects their intellectual property (IP)Creations of the mind, such as inventions; literary and artistic works; designs; and symbols, names and images used in commerce. Protected by patents, copyright and trademarks. and prevents competitors from easily copying their software.
  • Allows them to charge a fee for licenses, generating revenue.
  • Ensures quality control and compatibility as they control all modifications.
  • Can provide dedicated customer support and regular updates.

Off-the-Shelf vs. Bespoke Software

Two Approaches to Software Acquisition

Off-the-Shelf SoftwareSoftware created to be sold in bulk to many users such as Office or Adobe products. The manufacturer won't customise it for you.

This is general-purpose software designed to be sold to a large number of customers. It's ready to use "as is."

  • Examples: Microsoft Office, Adobe Photoshop, antivirus software, most video games.
  • Pros: Cheaper, readily available, well-tested, lots of support/tutorials.
  • Cons: May have many features you don't need, or lack specific features you do need; cannot be easily customized by the user.

Bespoke SoftwareSoftware that is specially designed and written for a single organisation or group of users. (Custom Software)

This software is specially designed and written for a specific organization or user to meet their unique requirements.

  • Examples: A custom inventory system for a specific warehouse, a unique patient management system for a hospital, specialized scientific analysis software.
  • Pros: Perfectly tailored to the user's needs, can provide a competitive advantage.
  • Cons: Very expensive, takes a long time to develop, may have limited support (reliant on original developers).

Legal Issues When Choosing Software

Important Considerations

When an organization chooses software, they must consider several legal aspects:

  • Abiding by the Software License: This is paramount. The license dictates how many users can use it, if it can be copied, modified, etc. Ignoring the license is a breach of contract and often copyright law.
  • Open Source Implications: If choosing open source software, the organization must understand the specific open source license terms.
    • They can usually make modifications.
    • However, some licenses (e.g., "copyleft" licenses like GPL) require that any modifications or derivative works are also made open source under the same terms.
    • They must usually credit all previous contributors if redistributing modified code.
  • Cost of Proprietary Software: Purchasing licenses for off-the-shelf proprietary software can be expensive, especially for many users. This needs to be budgeted for.
  • Data Protection Compliance: Any software that handles personal dataInformation relating to an identifiable living individual. Examples include name, address, email, medical records, online identifiers. must enable the organization to comply with the Data Protection Act (DPA)UK law governing the processing of personal data. It sets out principles for how organisations must handle individuals' information. (Often refers to DPA 2018 which incorporates GDPR).. This includes how data is stored, secured, and accessed.

Mark Scheme (1 mark for each aspect of a difference, max 2 differences for 4 marks):

Extension Activities

1. Specific Open Source Licenses

There isn't just one "open source license." Common examples include GPL (General Public License), MIT License, and Apache License.

Research Task: Choose two different open source licenses (e.g., GPLv3 and MIT). What are the key differences in their terms, particularly regarding modification and redistribution of the software and its derivatives?

2. Software Patents

While copyright protects the specific code (expression of an idea), patents can sometimes protect the underlying algorithms or methods used in software (the idea itself), though this is a complex and debated area.

Research Task: What is a software patent? How does it differ from copyright for software? What are some arguments for and against allowing software patents?

3. Digital Rights Management (DRM)

Proprietary software and media often use DRM technologies to control how they can be used, copied, or accessed.

Research Task: What is Digital Rights Management (DRM)? Give an example of how it's used. What are some ethical arguments for and against the use of DRM?
