Introduction to For Loops

IterationThe process of repeating a set of instructions. means repetition, and it's a fundamental concept in programming, allowing us to automate repetitive tasks efficiently.

for loops (as written in OCR Exam Reference Language) are a type of iteration structure used for definite iterationAlso known as count-controlled iteration. A loop that repeats a specific number of times, known before the loop starts., also known as a count-controlled loopA loop that repeats a predetermined number of times. FOR loops are count-controlled.. This means they are ideal when you know in advance exactly how many times you need a block of code to run.

We'll explore how for loops are structured in pseudocodeA plain language description of the steps in an algorithm or program, not tied to a specific programming language. OCR uses its Exam Reference Language (ERL) for this. (specifically OCR ERL) and see how they translate into a high-level language like Python. The loop variableA variable used in a loop to keep track of the current iteration. It often changes its value with each pass through the loop. is key to controlling these repetitions.

 Real-World Connection: For Loops in Action

                
            

            

                
for loops are workhorses in programming! They are used for:

                
    
                    
  • Processing items in a list or collection: e.g., calculating the total of items in an online shopping basket, sending a promotional email to every customer on a mailing list, checking each word in a document for spelling errors.
    • 
                    
  • Graphics and Games: e.g., drawing multiple similar objects on a screen (like stars in a night sky), updating the position of all enemies in a game a certain number of times per second, animating a character by cycling through a sequence of images.
    • 
                    
  • Data Analysis and Processing: e.g., reading through a specific number of lines in a data file, performing the same calculation on multiple data entries in a spreadsheet, generating a report for each month of the year.
    • 
                    
  • Repeating User Interface tasks: e.g., creating 10 buttons in a row on a webpage, populating a dropdown menu with a list of countries, displaying a set number of posts on a social media feed.
    • 
                

            

        


        

            

                
 Myth Busters: For Loop Edition

                
            

            

                

                    
Myth 1: "for loops can only count upwards."

                    
Reality: Not true! By using a step valueDetermines how the loop variable changes after each iteration. Default is usually 1. Can be negative for counting down. (e.g., step -1 in OCR ERL, or a negative third argument in Python's range() function), for loops can easily count downwards or even skip numbers (e.g., step 2 to count in twos).

                

                

                    
Myth 2: "The loop variableA variable used in a loop to keep track of the current iteration. It often changes its value with each pass through the loop. in a for loop must be called i or count."

                    
Reality: While i, j, k, or count are common conventions (especially for simple or nested loops), the loop variable can be any valid variable name. It's good practice to choose a descriptive name if it makes the code easier to understand (e.g., for studentNumber = 1 to 30).

                

                 

                    
Myth 3: "for loops are the only way to repeat code."

                    
Reality: for loops are for *definite* iteration (when you know the number of repeats). For *indefinite* iteration (when the loop continues as long as a condition is true, and you don't necessarily know how many times it will run), programmers use other loop structures like while loops or do...until loops (OCR ERL uses do...until, Python uses while True: ... if condition: break or similar for post-condition checks). We'll cover those in another lesson!

                

            

        


        

            

                
 Common Mistakes with For Loops

                
            

            

                

                    
Mistake 1: Off-by-One Errors

                    
Common Error: Wanting to loop 5 times and writing for i = 1 to 4 (only loops 4 times) or for i = 0 to 5 (loops 6 times). In Python, range(5) gives 0,1,2,3,4 (5 iterations), but range(1, 5) gives 1,2,3,4 (4 iterations).

                    
Correct Approach/Understanding: Carefully check your start and end values. OCR ERL's to is inclusive. If you want 5 iterations starting from 1, it's for i = 1 to 5. If starting from 0, it's for i = 0 to 4. Always trace a small example mentally or on paper.

                

                

                    
Mistake 2: Incorrect Step Value for Counting Down

                    
Common Error: Trying to count down, e.g., from 5 to 1, but forgetting to make the step negative: for x = 5 to 1 step 1. This loop won't run as expected because `x` increments and will never reach 1 by decreasing.

                    
Correct Approach/Understanding: When counting down, the step value must be negative, e.g., for x = 5 to 1 step -1.

                

                

                    
Mistake 3: Forgetting `next ` in OCR ERL

                    
Common Error: Writing the loop body but omitting the next i (or whatever the loop variable is) at the end of the loop in pseudocode.

                    
Correct Approach/Understanding: OCR ERL requires next  to signify the end of the loop body and to increment/decrement the loop variable. Python uses indentation for this, so it's a common point of confusion when switching between them.

                

                 

                    
Mistake 4: Confusing Loop Variable with Output

                    
Common Error: Intending to print the loop variable but printing a fixed string instead, or vice-versa. E.g., print("i") instead of print(i).

                    
Correct Approach/Understanding: Remember that print("text") prints the literal string "text", while print(variableName) prints the current value stored in `variableName`.

Key Takeaways

                
            

            

                
    
                    
  • for loops (OCR ERL) are used for definite iterationAlso known as count-controlled iteration. A loop that repeats a specific number of times, known before the loop starts. (count-controlled iteration).
    • 
                    
  • Key components include a loop variableA variable that controls the iterations of a loop. Its value typically changes with each iteration., a start value, an end value, and an optional step valueDetermines how the loop variable changes after each iteration. Default is usually 1. Can be negative for counting down..
    • 
                    
  • OCR ERL structure: for loop_variable = start_value to end_value [step step_value] ... next loop_variable.
    • 
                    
  • Python structure: for loop_variable in range(start, stop, step): ... (Note: stop value in Python's range is exclusive).
    • 
                    
  • for loops are essential for repeating code a known number of times or for iterating over sequences like characters in a string (using indexing in ERL) or items in a list.
    •
