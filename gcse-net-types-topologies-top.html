Introduction to Network Topologies

A network topologyThe physical or logical arrangement of nodes and connections within a network. refers to the layout or arrangement of the different devices (nodes) and connections on a network. It defines how devices are connected and how data flows between them.

There are several common topologies, each with its own strengths and weaknesses. The choice of topology depends on factors like the size of the network, budget, reliability requirements, and ease of management. In this lesson, we will focus primarily on two important topologies: StarA LAN configuration in which a central node (e.g., a switch or hub) controls all message traffic. All devices connect directly to this central node. and MeshA network setup where devices are interconnected with many redundant interconnections between network nodes. In a full mesh, every node is connected to every other node..

Real-World Topologies

  • Star Topology: Very common in modern LANs. Most home and office networks use a star topology with a central switch or router. Ethernet LANs are typically implemented as star topologies.
  • Mesh Topology:
    • Full Mesh: Rarely used for entire LANs due to cost and complexity, but might be used for critical backbone connections between important servers or routers.
    • Partial Mesh: More common. Some important devices might have multiple connections for redundancy, while less critical devices have fewer. The internet itself can be seen as a massive partial mesh network of routers.
    • Wireless Mesh Networks (WMNs): Increasingly popular for providing Wi-Fi coverage over large areas (e.g., city-wide Wi-Fi, large outdoor events, smart home devices). Nodes relay data for each other.
  • Bus and Ring Topologies (Older): While less common now for new LAN installations, understanding them provides historical context. Bus topologies were used in older Ethernet versions (e.g., 10BASE2 with coaxial cable). Ring topologies were used in systems like Token Ring.

Topology Myth Busters!

Myth: "A Star topology always uses a hub as its central device."

Reality: While older Star networks might have used hubs, modern Star networks almost exclusively use switchesA network device that connects devices on a LAN and uses MAC addresses to forward data intelligently to the intended recipient. as the central device. Switches are more efficient as they send data only to the intended recipient, unlike hubs which broadcast to all connected devices.

Myth: "Mesh networks are always wireless."

Reality: Mesh topologiesA network setup where devices are interconnected with many redundant interconnections between network nodes. can be implemented with wired or wireless connections. While wireless mesh networks (WMNs) are becoming popular for extending Wi-Fi coverage, wired mesh networks are also used, especially for critical infrastructure requiring high reliability (e.g., connecting core routers in an ISP's network).

Myth: "In a Star network, if the central switch fails, devices can still communicate directly with each other."

Reality: This is false. The central device (switch or hub) in a Star topologyA LAN configuration in which a central node (e.g., a switch or hub) controls all message traffic. All devices connect directly to this central node. is a single point of failure. If it fails, all communication between the connected nodes stops. Devices in a Star network do not have direct connections to each other.

Exam Practice Questions

1. Describe one advantage of using a star network topology compared to a bus topology. [1 mark]

Mark Scheme (Any one from):

  • If one cable fails (in a star), only that single node is affected (not the whole network).
  • It is easier to add new nodes/devices (without disrupting the network).
  • Fewer data collisions (especially if a switch is used at the center).
  • Easier to identify faults.

2. Explain why a mesh network topology might be chosen for a network that requires high reliability. [2 marks]

Mark Scheme:

  • A mesh network has multiple paths between nodes / redundant connections. (1 mark)
  • If one path or node fails, data can be rerouted through alternative paths, so the network can continue to operate. (1 mark)
