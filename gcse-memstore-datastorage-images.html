Introduction to Digital Images

Digital images are everywhere, from photos on your phone to graphics on websites. The most common way computers store these images is using a bitmapAn image made of pixels, it blurs when scaled up, used for photos format.

Imagine an image as a grid. Each tiny square in this grid is called a pixelA bitmap image is broken up into a grid of these, it's short for "picture element". Each one has a single colour which is encoded in binary. (short for "picture element"). The computer stores the colour of each pixel as a binary number. The overall image is simply a collection of all these pixel colour codes, stored in sequence.

Two key factors determine an image's appearance and file size: resolutionThe number of pixels in an image, higher value means a more detailed image but higher file size. The area is defined by the image width and height in pixels e.g. 1920x1080. and colour depthThe number of bits used to store the colour for each pixel, a higher number means more variety of colours.. We'll explore these in this lesson!

Real-World Context

Understanding how images are stored is crucial in many fields:

  • Web Design: Choosing the right image resolution and colour depth is vital for balancing visual quality with fast loading times on websites.
  • Digital Photography: Photographers work with RAW image files (which store more data) and compressed formats like JPEG. Understanding file size implications is key for storage and sharing.
  • Medical Imaging: X-rays, CT scans, and MRIs are digital images. High resolution and appropriate colour/grayscale depth are essential for accurate diagnosis. File sizes can be very large.
  • Graphic Design & Printing: Designers need to understand resolution (often measured in DPI - Dots Per Inch for print) to ensure images look sharp when printed.
  • Game Development: Game textures are bitmap images. Optimizing their size and format is crucial for game performance and storage space.

Image Myth Busters!

Myth: "You can make a small, blurry image super sharp just by increasing its resolution in software."

Reality: Increasing the resolution of a bitmap image (upscaling) doesn't add new detail that wasn't there. The software essentially guesses or duplicates existing pixelsA bitmap image is broken up into a grid of these, it's short for "picture element". Each one has a single colour which is encoded in binary., which can make the image look larger but often more pixelated or blurry. True detail comes from the original capture resolution.

Myth: "More colours always means a drastically better image."

Reality: While a higher colour depthThe number of bits used to store the colour for each pixel, a higher number means more variety of colours. allows for more colours and smoother gradients, the human eye can only distinguish a certain range. For many images, 24-bit colour (millions of colours) is sufficient. Using 32-bit or higher might not offer a noticeable visual improvement for general viewing but will significantly increase file sizeThis goes up if we make the quality better by increasing the colour depth or resolution, meaning we need more storage.

Exam Practice Questions

1. An image is 100 pixels wide and 50 pixels high. Each pixel uses 1 bit for its colour. Calculate the file size of this image in Bytes. Show your working. [3 marks]

Mark Scheme:

  • Total pixels = Width × Height = 100 × 50 = 5000 pixels (1 mark)
  • File size in bits = Total pixels × Colour depth = 5000 × 1 = 5000 bits (1 mark)
  • File size in Bytes = File size in bits / 8 = 5000 / 8 = 625 Bytes (1 mark)

2. Explain what is meant by 'metadata' in the context of an image file. Give one example of image metadata. [2 marks]

Mark Scheme:

  • Metadata is data about the image file itself / information about the image rather than the image content (1 mark).
  • Example: Image resolution / dimensions (width & height) / colour depth / date created / camera settings / GPS location / file type. (1 mark for one valid example).

Key Takeaways

  • Digital images are often stored as bitmapsAn image made of pixels, it blurs when scaled up, used for photos, which are grids of pixelsA bitmap image is broken up into a grid of these, it's short for "picture element". Each one has a single colour which is encoded in binary..
  • Each pixel's colour is represented by a binary code.
  • ResolutionThe number of pixels in an image, higher value means a more detailed image but higher file size. The area is defined by the image width and height in pixels e.g. 1920x1080. (width × height in pixels) determines the number of pixels and thus the image detail.
  • Colour DepthThe number of bits used to store the colour for each pixel, a higher number means more variety of colours. (bits per pixel) determines how many colours can be represented (e.g., 1-bit = 2 colours, 8-bit = 256 colours, 24-bit = millions of colours).
  • Image file size = (Width × Height) × Colour Depth (in bits). Divide by 8 for Bytes.
  • Higher resolution and higher colour depth lead to better qualityIncreasing the colour depth or resolution will make this better images but also larger file sizesThis goes up if we make the quality better by increasing the colour depth or resolution, meaning we need more storage.
  • MetadataData about data, such as image dimensions, resolution, colour depth, date created, file type. is data stored with the image file that describes the image (e.g., dimensions, colour depth, date taken).
