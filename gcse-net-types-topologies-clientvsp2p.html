Introduction to Network Models

Beyond just connecting computers, how those computers interact and share resources is defined by the network model. The two main models we'll look at are Client-ServerA network model where client computers request services/resources from a central server computer. and Peer-to-Peer (P2P)A network model where all computers have equal status and can share resources directly with each other without a central server..

A LAN (Local Area Network)Computers are connected to each other and use MAC addresses to communicate. Restricted to a small geographical area / site / other suitable example. Hardware (routers,WAPs, switch) owned by the company/home owners. typically covers a small geographical area, like a single building or campus (e.g., your school network). The organization usually owns the network hardware.

A WAN (Wide Area Network)Network that spans a large geographical area. Communication medium (cables that connect networks together) is owned by telecommunications companies e.g. BT. spans a large geographical area, often connecting multiple LANs together (e.g., the internet, or a company's network connecting offices in different cities). The infrastructure (like undersea cables) is often owned by telecommunications companies.

Both LANs and WANs can be structured using either the client-server or peer-to-peer model, though client-server is more common for larger, managed networks.

Real-World Context

  • Client-Server Networks: Common in businesses, schools, and organizations. Examples include web servers hosting websites, file servers storing shared documents, email servers, and database servers. The internet itself largely operates on a client-server model (your browser is a client requesting pages from web servers).
  • Peer-to-Peer (P2P) Networks: Used in small office/home office (SOHO) setups for simple file and printer sharing. Also known for file-sharing applications (like BitTorrent, though these have legal implications if used for copyrighted material). Some modern communication apps might use P2P for direct connections once established.
  • LANs: Your home Wi-Fi network, the network in your school's computer lab, or a small office network.
  • WANs: The internet is the ultimate WAN. Companies with multiple branches across different cities or countries use WANs to connect their LANs.

Network Model Myth Busters!

Myth: "Peer-to-Peer networks are completely insecure and only used for illegal activities."

Reality: While some P2P file-sharing applications have been associated with copyright infringement, the P2P model itself is a legitimate network architecture. It can be very useful for simple, direct sharing in trusted environments (like a home network). Security is managed by individual peers, which can be a drawback in larger or less trusted settings, but it's not inherently illegal.

Myth: "You always need an expensive, dedicated server machine for a client-server network."

Reality: While large organizations use powerful, dedicated server hardware, a "server" is fundamentally defined by its role – providing services. In a very small setup, a regular desktop computer could be configured to act as a simple file or print server for a few clients. However, for reliability and performance with many users, dedicated server hardware and software are recommended.

Myth: "LANs can't connect to the internet."

Reality: Most LANs are connected to the internet (a WAN) via a router. The router acts as a gateway, allowing devices on the LAN to access resources on the WAN and vice-versa, while also often providing security features like a firewall.

Exam Practice Questions

1. Describe two differences between a LAN and a WAN. [2 marks]

Mark Scheme (Any two from):

  • LAN covers a small geographical area (e.g., single site/building) (1) vs. WAN covers a large geographical area (e.g., countries) (1).
  • LAN infrastructure is typically owned by the organization (1) vs. WAN infrastructure often uses third-party carriers (e.g., telecommunication companies) (1).
  • LANs usually have higher data transfer speeds / lower latency than WANs (due to shorter distances and dedicated infrastructure) (1).
  • (Allow other valid differences like connection methods typically used, scale of users etc.)

2. A small business has 10 employees who need to share files and a central printer. They are considering setting up a network.

(a) Identify which network model, client-server or peer-to-peer, would be more suitable. [1 mark]

(b) Justify your choice for part (a), giving two reasons. [2 marks]

Mark Scheme:

(a) Client-Server (1 mark). (Allow Peer-to-Peer if well justified in part b, though Client-Server is generally better for this size with central needs).

(b) Justification for Client-Server (any two from):

  • Centralised file storage is easier to manage and back up. (1)
  • Centralised printer management is simpler. (1)
  • Easier to implement central security policies / user authentication. (1)
  • More scalable if the business grows. (1)

OR Justification for Peer-to-Peer (if chosen, needs strong argument for a business context):

  • Lower initial cost / no need for a dedicated server. (1)
  • Simpler to set up for a very small number of users if technical expertise is limited. (1)
  • Less reliant on a single server (if server fails in C-S, network services stop). (1)

Key Takeaways

  • LAN (Local Area Network)Computers are connected to each other and use MAC addresses to communicate. Restricted to a small geographical area / site / other suitable example. Hardware (routers,WAPs, switch) owned by the company/home owners.: Covers a small geographical area (e.g., building). Typically owned by the organization.
  • WAN (Wide Area Network)Network that spans a large geographical area. Communication medium (cables that connect networks together) is owned by telecommunications companies e.g. BT.: Spans a large area (e.g., cities, countries). Often uses third-party infrastructure. The Internet is a WAN.
  • Client-Server ModelA network that uses central computers, known as servers to enable resource sharing and to facilitate communication between the other computers on the network.: Clients request services from a central server. Benefits: centralized management, security, backups. Drawbacks: server dependency, higher cost.
  • Peer-to-Peer (P2P) ModelNo server. Computers are directly connected to each other. Computers are independent. Computers will have software installed/updated individually (no central installation/updates). Computers will need own security // no central security. Computers will have their own files // no central file storage.: All devices (peers) have equal status and share resources directly. Benefits: easy setup, lower cost, no single point of failure. Drawbacks: decentralized security and backups, can be slower with many users.
  • The choice of model depends on the network's size, budget, security needs, and required services.
